Introduction

Scutum Admin is a professional HTML5/CSS3 Material Design template based on UIkit Framework (v3).

It comes with 2 main versions: HTML and VueJS that can be used to build any backend application: shop backend, anylytics backend, CMS, CRM etc.

HTML version source code is build with the help of PHP language (includes, simple functions and helpers).
VueJS version is based on Nuxt.js framework, universal mode (SSR + SPA) is set as default but can easily be switched to SPA mode.

This template has been built with the help of the following tools

  • Node.js open-source, cross-platform JavaScript run-time environment that executes JavaScript code outside of a browser
  • Yarn dependency manager
  • Gulp 4 streaming build tool - mainly HTML version

Config

In scutum.config (/assets/js/scutum_admin.js) you can change this settings:

Async loader

LoadJS is a tiny async loading library for modern browsers (IE9+).
It has a simple yet powerful dependency management system that lets you fetch JavaScript, CSS and image files in parallel and execute code after the dependencies have been met.

To improve the performance and to load assets only when they are needed LoadJS plugin has been used.

There is a require-like dependency manger implemented in Scutum Admin

/**
 * Require-like dependency manager
 * @param {array} 		bundleIds	bundle names (defined in scutum.bundles)
 * @param {function} 	successFn	function to run after all assets has been successfully loaded
 * @param {boolean} 	async		load files synchronously or asynchronously
 */
 scutum.require(bundleIds, successFn, async);

By default all assets are loaded asynchronously but you can load them in series, to do that set async option to false.

In assets/js/scutum_common.js you can find predefined bundles that are used in Scutum Admin.

scutum.bundles = {
    'bundle name': ['path to js file', 'path to css file'],
    ...
}

How to use it?

Below you can find some examples how to use async loader.

// datepicker
scutum.require(['flatpickr'], function () {
    // initialize datepicker
})

// billboard.js charts (d3 must be loaded before billboard.js)
scutum.require(['d3', 'billboard'], function () {
    // initialize chart
}, false)

// calendar (plugins must be loaded synchronously)
scutum.require(['tui-snippets', 'tui-timepicker', 'tui-datepicker', 'tui-calendar', 'handlebars'], function () {
    // initialize calendar
}, false)

Get Started

Before you can start to customize Scutum Admin template you must download and install Node.js (cross-platform runtime environment), Yarn (dependency manager) and Gulp 4 (automated tasks).

If this tools are installed and working correctly you can run this command from Scutum root directory (/admin/html/src) to install all dependencies and regenerate css/js files:

$ yarn sc-setup

Now if dependencies and plugins were installed without errors you can use various gulp tasks to customize Scutum Admin template.

Gulp Tasks

Scutum Admin uses Gulp 4 to automate tasks like compiling SCSS to CSS, minifing JS files, live reloading assets and HTML.

If you don't have Gulp 4 installed globally you can use locally installed gulp.
To do that use yarn gulp 'task name' command eg. yarn gulp sass:main 

Available tasks (/build/gulp-tasks/*.js)

SASS (sass.js)
    gulp sass:main - compile main stylesheet (assets/css/scutum.css|scutum.min.css)
    gulp sass:plugins - compile stylesheets for plugins
    gulp sass:error_page - compile error page stylesheet (error_page.css/error_page.min.css)
    gulp sass:login_page - compile login page stylesheet (login_page.css/login_page.min.css)
    gulp sass:themes - compile themes (assets/css/themes/theme_*.css) and concatenate all themes (themes_combined.min.css)
    gulp sass:style_switcher - compile style switcher stylesheet (style_switcher.css/style_switcher.min.css)

JS (js.js)
    gulp js:vendor - common plugins used in Scutum template
    gulp js:uikit - custom build of UIkit components
    gulp js:minify - minify custom/common scripts

SERVE (serve.js)
    gulp serve - synchronised browser testing - https://www.browsersync.io/docs/

BUILD (build.js)
    gulp build - build dist folder (minified js/css/json and HTML files, require PHP and some customization in build/generate_html.php)

Development server (Browsersync)

Web server with installed PHP is required to run Browsersync.

To help with the live reload assets/HTML content, automated testing and auto compiling Sass Browsersync plugin has been used.

Before you can run Browsersync you need to change host and proxy settings in /build/gulp-tasks/serve.js

proxy option is required because Scutum Admin use PHP (it needs to be installed on web server).

host option can be deleted/commented out if your web server is installed locally.

Now you can run following command to start Browsersync:

$ yarn sc-dev

Commands

yarn sc-setup Install all dependencies and process sass/js files.
yarn sc-dev Launch development server (live reload assets/HTML, auto compile sass etc.) on 192.168.1.188:3101
yarn sc-update Remove node_modules directory and update dependencies.
yarn sc-cleanup Remove node_modules.
yarn gulp Run locally installed Gulp 4.

VueJS version

Introduction

Scutum Admin Vue.js version is based on Nuxt.js framework.

Nuxt.js is a framework for creating Vue.js applications, you can choose between Universal, Static Generated or Single Page application.

Default mode used in Scutum Admin is 'universal' but can be easily changed to 'spa' or you can generate static pages that you can host on any static hosting.

Structure

Scutum Admin utilize default folder structure recomended by Nuxt.js

/ Scutum Admin root
/assets The assets directory contains your un-compiled assets (Sass files, images, fonts etc.)
/build The build directory contains gulp tasks that you can use to create custom version of UIkit JS or to minify js files.
/components The components directory contains your Vue.js Components.
/data The data directory contains json files used in applcation (sidebar menu, fake data etc.).
/layouts The layouts directory includes your application layouts.
/middleware The middleware directory contains your Application Middleware. Middleware lets you define custom functions that can be run before rendering either a page or a group of pages (layouts).
/pages The pages directory contains your Application Views and Routes.
/plugins The plugins directory contains your Javascript plugins that can be run before instantiating the root Vue.js application.
You can also import them directly in pages/components.
/static The static directory is directly mapped to the server root and contains files that likely won't be changed (i.e. the favicon)
/store The store directory contains your Vuex Store files.

Get Started

Before you can start working with this version you need to install Node.js and Yarn (dependency manager)

If this tools are installed and working fine you can run this command from Scutum root directory (/admin/vuejs/src) to install all dependencies:

$ yarn sc-setup

Now if there were no errors you can start customizing Scutum Admin template.

Configuration

As mentioned before, Scutum Admin Vue.js version is based on Nuxt.js framework, all configuration options you can change/adjust in nuxt.config.js.

Here you can find a configuration guide: Nuxt.js configuration

Modes

To change mode property open /nuxt.config.js and set desired mode.
Available modes: 'universal' (default) and 'spa'.

Here you can find more info about modes:

The mode Property
Server Rendered (Universal SSR)
Single Page Applications (SPA)

Static Generated (Pre Rendering)

Scutum Admin offers static site generator out of the box (thanks to Nuxt.js framework), you can easily pre-render static site using a single command:

$ yarn generate

When you run this command Nuxt.js will generate web application that you can host on any static hosting.

Default directory name (dist) for static generated web aplication is scutum-universal (when mode is set to universal)
you can change that directory name in nuxt.config.js

Application generated by this command must be served from /root directory,
if you want to serve it from subfolder (dist) you need to set serveFromSubFolder to true in nuxt.config.js

Laravel integration

In admin/vuejs/laravel-nuxt/ you can find files that will help you integrate Nuxt.js with Laravel. In that integration Laravel will work as an API and Nuxt.js as a Single Page Application.

Here are the steps to install Laravel and integrate Nuxt.js with it:

1. Create Laravel project ('laravel' is the default project name but you can use any name, just replace laravel with your project name)

composer create-project laravel/laravel laravel 5.8 --prefer-dist

2. Go inside your new project

cd laravel/

3. Install laravel-nuxt (for PHP) 

composer require pallares/laravel-nuxt

4. Create 'nuxt' folder in laravel/resources/ and copy assets, components, data, layouts, middleware, pages, plugins, static, store folders from scutum/admin/vuejs/src/ to laravel/resources/nuxt/

5. Copy files from scutum/admin/vuejs/laravel-nuxt/ to laravel/ (replace files)

6. Install all dependencies

yarn sc-setup

7. Run dev server (local)

yarn start

7a. Run dev server (remote)

yarn start --hostname remote-server-ip

8. Deploy application 

yarn build

8a. After build copy files/folder from laravel/resources/nuxt/static/ to laravel/public/

When you run dev server you can go to /api route to check simple API call from Laravel.

Integration based on Create a SPA with Laravel and Nuxt

After customizations you can use gulp tasks to compile SCSS files to CSS (HTML version), check available SASS tasks

Themes

In /assets/scss/themes folder you can find few predefined themes (theme_a...theme_h) and two themes that can be used as a base:

user_theme_md.scss - material design colors

user_themes.scss - hex colors

HTML version

After customization you need to run yarn gulp sass:themes to compile your theme to CSS (assets/css/themes/user_theme.css and assets/css/themes/user_theme_md.css)

Now when you have yor custom theme you need to include it in your app, to do that open index.php, paste loadjs('assets/css/themes/theme_user.css', {}); after main stylesheet and add class sc-theme-user to <html> element.

Vue.js version

To activate your theme, open /layouts/default.vue file and uncomment code in mounted hook.
